# OSQuery Basics

OSQuery allows operating system information to be queried using SQL syntax.

Instead of manually checking logs and system utilities separately, OSQuery exposes endpoint information as structured tables.

Examples include:

* processes
* users
* listening_ports
* startup_items
* logged_in_users

This approach simplifies endpoint investigations and threat hunting operations for security teams.

OSQuery is commonly used for:

* Incident response
* Threat hunting
* Endpoint visibility
* Security monitoring
* Compliance validation
